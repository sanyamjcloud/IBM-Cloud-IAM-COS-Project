# IBM Cloud IAM: Secure Access Control for Cloud Object Storage (COS)

## Project Overview

This project demonstrates how secure access control is implemented using Identity and Access Management (IAM) for Cloud Object Storage. It focuses on configuring users, assigning roles, and validating permissions to ensure only authorized access to storage resources.

---

## Objectives

* Configure secure cloud storage environment
* Implement role-based access control
* Validate user permissions through testing

---

## Tools & Platforms Used

* IBM Cloud Platform
* Cloud Object Storage (COS)
* Identity & Access Management (IAM)
* Web Console

---

## Key Features

* Access group creation for centralized permission management
* Role assignment to control user actions
* Secure file upload validation
* Authorization testing with invited users

---

## Implementation Steps

1. Created Cloud Object Storage instance
2. Configured bucket settings
3. Uploaded test file
4. Created access group
5. Assigned roles to group
6. Added user to group
7. Verified permissions via testing

---

## Result

Secure access control successfully implemented. Only authorized users were able to perform permitted operations, confirming correct IAM policy configuration.
